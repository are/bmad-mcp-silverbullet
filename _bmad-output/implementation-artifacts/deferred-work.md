## Deferred from: code review of story-1-10-read-side-tool-handlers-list-pages-search-pages-read-page (2026-05-01)

- **Per-row drop hardening for non-string `page.ref` / `hit.ref`** ‚Äî The defensive re-validation loop catches `RefValidationError` only via `instanceof`. A non-string ref returned by SB (e.g. `42`, `null`) reaches `makeRef` ‚Üí `Buffer.byteLength(non-string, 'utf8')` throws `TypeError`, which is NOT a `RefValidationError`, so `throw refErr` fires and aborts the entire call instead of dropping the one bad row per AR10's intent. Spec Task 3 codified the `throw refErr` pattern, so this is spec-faithful ‚Äî but tightening to `catch (refErr) { ctx.logger.warn(...); continue; }` (warn-and-continue for any throw, not just RefValidationError) better matches AR10. Revisit when 2.x handlers land or when an SB upgrade is found returning non-string refs. (`src/mcp/handlers/list-pages.ts:101`, `src/mcp/handlers/search-pages.ts:100`)
- **Top-level shape validation of SB-returned arrays** ‚Äî `for (const page of raw.pages)` / `for (const hit of raw.hits)` produces an opaque iterator `TypeError` if SB returns a malformed shape (missing/null `pages`/`hits`, e.g. after a Lua refactor). The catch arm correctly converts to `infrastructure_error`, but `details.underlying` is generic JS rather than "SB returned malformed list-pages payload". Adds operator-side observability without changing the agent-visible result. Patch shape: `if (!Array.isArray(raw.pages)) throw infrastructureError(new Error('SB returned malformed list-pages payload'));`. (`src/mcp/handlers/list-pages.ts:91`, `search-pages.ts:89`)
- **Unbounded `q` length on `search_pages`** ‚Äî `z.string().min(1)` has no upper bound; a multi-MB query string is logged verbatim into the audit `args` field per AR31. Audit-log DoS / forensic-noise vector. Add `z.string().max(N)` (e.g., 1024 or 4096 chars) once an operator-facing length budget is decided, OR sanitize the audit `args` projection separately. (`src/mcp/handlers/search-pages.ts:34`)
- **Unbounded list/search response payload size** ‚Äî `formatToolSuccess(JSON.stringify({ pages: visibleRefsPlain }))` and the search-hits equivalent emit a single text-content block with no size cap. A 10k-page space (or 10k search hits) produces a multi-MB payload that may breach MCP transport limits and is fully buffered in memory before return. Pagination is a Growth-tier concern (HTTP/SSE transport) but a soft truncation guard ("returning first N of M; refine your query") would prevent transport-level opacity. (`src/mcp/handlers/list-pages.ts:116`, `search-pages.ts:122`)
- **Three near-duplicate `summarizeZodIssues` helpers** ‚Äî Each handler ships its own ~15-line copy with cosmetic per-tool message tweaks. Spec dev-notes already deferred a `tests/integration/_helpers.ts` extraction until the 5th duplication (when 2.x lands); the same logic applies to `summarizeZodIssues`. Extract to `handler-template.ts` parameterised by tool name when the 5th handler arrives (Story 2.2's `append_to_page` would be #4). (`src/mcp/handlers/list-pages.ts:40-54`, `search-pages.ts:37-53`, `read-page.ts:35-51`)
- **`isDomainError` accepts unknown `reason` values** ‚Äî Structural guard checks `typeof reason === 'string' && typeof details === 'object'` only; doesn't validate `reason` is one of the known `ReasonCode` literals. AC7 mandates exactly this structural test, so the implementation is faithful ‚Äî but a buggy upstream that throws `{ reason: 'kaboom', details: {} }` would be silently re-emitted as a "DomainError" without `formatToolError` having a template. Tighten by validating `reason` against the `ReasonCode` union if a real bug surfaces, or document the trust assumption explicitly. (`src/mcp/handler-template.ts:182-189`)
- **`access === 'none'` gate brittle to future AccessMode additions** ‚Äî `read-page.ts:106` uses `if (access === 'none')` to short-circuit. Robust today (AccessMode is the closed `'none' | 'read' | 'append' | 'write'` set), but a future mode (e.g., `'list-only'` or `'restricted-read'`) added without `'none'`-equivalent semantics would silently fall through to the SB read. (`src/mcp/handlers/read-page.ts:106`, `src/permissions/access-mode.ts`)

## 2. What worked — patterns to repeat in Epic 2

1. **Handler-template shape (Story 1.10).** The canonical `parse → permission → execute → respond` shape with `finally`-block audit and `HandlerContext` injection threaded through 1.11 wiring with zero rework. Epic 2's four write-side handlers (`append_to_page`, `edit_page`, `create_page`, `delete_page`) inherit this shape unchanged.
2. **Factory + closure for stateful modules.** `createLogger` (1.3), `createAuditLogger` (1.5), `createFreshnessState` (1.9) — three consumers, zero protocol mismatches. No `class`, no module-level singletons. Use the same shape for any new stateful module in Epic 2.
3. **`scrubSecrets` (1.6) as the canonical untrusted-input → safe-output pattern.** Recursive descent + `WeakSet` cycle-guard + sentinel fallbacks (`<invalid-date>`, `[object Object]`) for non-standard inputs. Already referenced again in 1.13's threat-model assurance.
4. **LRU via `Map.delete-then-set` (1.9).** Pinned the JS idiom for cache-like structures; relies only on `Map`'s insertion-order contract.
5. **Latency-first instrumentation (1.7).** Shipping the latency harness inside Story 1.7 (rather than as a "later optimization story") gave the team budget headroom data before the read handlers landed. Carry this discipline into Epic 2 — measure before optimizing.
6. **Stdio-discipline smoke test (1.12).** Catches what the lint rule cannot (transitive-dep `console.log`, accidental `process.stdout.write`). Already a known gap that it only exercises `list_pages`; broadening is on the action list below.

## 3. Challenges — recurring themes from review cycles

These are the four patterns that surfaced in ≥2 stories. They are the load-bearing input to Epic 2's pre-handler checklist (action item A1).

1. **Stream-lifecycle edge cases** (1.3, 1.5, 1.11). Synchronous throws inside `write()` / `flushQueue()` bypassed fire-and-forget boundaries; missing `'error'` listeners on stdin/stdout/audit-stream let broken pipes escalate to `uncaughtException`. Fix pattern: wrap streams (`safeWrite`, `treatAsErrored`); install error listeners *before* `server.connect()`. Story 1.11 specifically had to refactor signal-handler installation to wrap earlier.
2. **Type-narrowing complexity under strict TS flags** (1.2, 1.6, 1.8). `noUncheckedIndexedAccess` + `exactOptionalPropertyTypes` forced exhaustive guards. Story 1.8 took 4 review patches to land access-mode ordering tests cast-free. The flags are correct; the lesson is that narrowing patches accumulate when not designed up-front (discriminated unions, `Number.isInteger`, post-iterator `!== undefined`).
3. **Boundary-validation gaps** (1.4, 1.6, 1.10). Non-Error objects, malformed SB payloads, non-string refs slipped through as generic `TypeError`. The fix is a layered narrowing rule: zod at the MCP surface, domain primitives (`Ref`) at module boundaries, explicit shape narrowing for runtime-unknown SB responses (`isDomainError`, per-row `makeRef` re-validation, scrubber DAG/Invalid-Date guards).
4. **Doc-vs-code drift in 1.12 and 1.13.** Threat-model overstated lint scope (`console.log` is lint-blocked; `process.stdout.write` is gated by the smoke test, not lint). Permissions guide overstated "takes effect immediately" (SB tag indexer is async). All caught in review with no story delay, but the same drift surface will re-appear when `docs/audit-log.md` (2.7) and `starter-template.md` (2.8) land.

## 4. Cross-story dependency surprise

Story 1.4 (config) needed `Result<T, E>` (two-param), but 1.2 had shipped `Result<T>` (one-param, locked to `DomainError`). Resolved via D1 widening to `Result<T, E = DomainError>` — a surgical retrofit, no 1.2 reopening. **General lesson:** pure-domain primitives that ship without a live consumer ship under-generalized. Captured as action item A4.

## 5. Single biggest insight

> Architecture was sound from Story 1.2 onwards. The post-implementation brittleness came entirely from **stream-lifecycle edge cases** and **doc-vs-code drift** — not from architectural error.

A pre-write checklist ("error listeners on every stream? try/catch covers clock/iterator throws? doc claims match what code enforces?") would have caught most of the rework patches in 1.3, 1.5, 1.11, 1.12, and 1.13. Action item A1 codifies it.
